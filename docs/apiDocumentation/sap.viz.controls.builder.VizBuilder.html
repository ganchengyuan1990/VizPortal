Class: VizBuilder

Extends: sap.viz.controls.common.VizControlHost

Constructor

VizBuilder (div, uiConfig)

The VizBuilder is a composite control of following sub controls of Viz Container. VizBuilder can be customized to show or hide each sub control.
    1. SwitchBar: is a control that categories all viz types in groups. User can click on the arrow icon of group to display chart type items, click one of these items to change the current viz type.
    2. FeedingPanel: is a control that lists feeding containers of chart feeds and offers ui interactions about feeds such as open a feeding list, add and delete feed items.
    User can open a feeding list by clicking the '+' icon at end of the feeding container; Add a feed item to the feed containers by dragging and dropping an object into the feeding container area or selecting a feed item of the feeding list;
    Delete a feed item from the feed containers by dragging and dropping an object outside the feeding container area or clicking its 'Ã—' icon after mousing over it.
    3. MapBuilder: is a control that lists feeding containers of geo map.
Please refer to sap.viz.controls.switchbar.SwitchBar, sap.viz.controls.feeding.FeedingPanel and sap.viz.controls.mapbuilder.MapBuilder classes for details.
Parameters:
  • div : HTMLElement
    • The dom element to host the control.
  • uiConfig : JSON [Optional]
    • Configuration for the control. If without the uiConfig, use the default uiconfig to create the control.
                uiConfig.defaultVizType:           Set the default viz type
            uiConfig.header.visible:           Show/hide the text of selected chart type. The default value is true.
            uiConfig.controls:                 Sub controls group
            uiConfig.controls.switchBar:       Configuration for SwitchBar
            uiConfig.controls.feedingPanel:    Configuration for FeedingPanel
            uiConfig.controls.mapBuilder:      Configuration for MapBuilder
    Sample code:
        // Get a default uiConfig of VizBuidler from GlobalConfig. 
    var uiConfig = sap.viz.controls.common.config.GlobalConfig.defaultUIConfig(sap.viz.controls.common.config.GlobalConfig.DEFAULT_UICONFIG_TYPE_BUILDER);
    // Customize the default uiconfig. For example: Hide the header "TYPE: Column Chart".
    uiConfig.header.visible = false;
    // Create a VizBuilder control.
    var vizBuilder = new VizBuilder($(div), uiConfig);

Events

requestFeed

Dispatches "requestFeed" event when click '+' icon at right of the feeding container and the flag useBuildInFeedList of uiConfig values false, which means application does not use Viz Container's built-in feedlist.
Then application may need to pop up a customized feedlist, and dispatch a event with feed when a item selected.
Please ignore this event if using Viz Container's built-in feedlist.
type: "requestFeed"
handler: function({
                   type: "requestFeed",
                   data: {
                       feedId: "",
                       fieldLocalPosition: {shelfWidth, x, y}
                   }
})
Properties
  • evt.type : String
    • "requestFeed"
  • evt.data.feedId : String
    • "feedItem's id means which feeding container will be feeded"
  • evt.data.fieldLocalPosition : Object
    • "the data for calculating the location where the feed list will pop up at"
    Properties
    • shelfWidth : Object
      • "the width of a feeding shelf"
    • x : Object
      • "the left position of a feeding shelf"
    • y : Object
      • "the top position of a feeding shelf"
Example

Sample code:

var uiConfig = sap.viz.controls.common.config.GlobalConfig.defaultUIConfig(sap.viz.controls.common.config.GlobalConfig.DEFAULT_UICONFIG_TYPE_BUILDER);
uiConfig.controls.feedingPanel.useBuildInFeedList = false;
var vizBuilder = new VizBuilder($(div), uiConfig);
var feedRequestHandler = function(event) {
    //User needs to build up a feedlist
    var feedList = new FeedList(...);
    //User needs to add item selected event for feedlist and handle it
    feedList.addEventListener('itemSelected', function(event) {
        var selectedItem = event.data.selectedItem;
        //Set back feeds by calling vizBuilder.feeds() API
        vizBuilder.feeds(selectedItem)
    }, this);
    //User needs to calculate the position feedlist pop up at.
    var popupPosition = feedList.calculatePopupPosition(event.fieldLocalPosition);
    feedList.open(popupPosition);
}
vizBuilder.addEventListener('requestFeed', feedRequestHandler);

feedsChanged

Dispatches "feedsChanged" event when the feeding changes due to add/remove/change FeedItems on feeding panel
Application should listen to "feedsChanged" event to get the corresponding data and set it back to VizFrame to to update the visualization with the new data. 
type: "feedsChanged"
handler: function({
                   type: "feedsChanged",
                   data: {
                       feeds: {Array}
                   }
})
Properties
  • evt.type : String
    • "feedsChanged".
  • evt.data.feeds : Array<sap.viz.controls.common.FeedItem>
Example

Sample code:

var vizBuilder = new VizBuilder(...);
var feedsChangedHandler = function(event){
    var feeds = event.data.feeds;
    console.log(feeds);
}
vizBuilder.addEventListener('feedsChanged', feedsChangedHandler);

actionChanged

Dispatches "actionChanged" event when click setting icon of measure feed item that showed by mouse over the feed item, to pop up a action list and select one item of the list.
The application can handle something listed on the list about changing the the current data by listening to this event, such as sorting by the current measures of data in ascending or descending order. 
type: "actionChanged"
handler: function({
                   type: "actionChanged",
                   data: {
                       feedId,
                       analysisObjectID,
                       actionID
                   }
})
Properties
  • evt.type : String
    • "actionChanged"
  • evt.data.feedItemID : String
    • Which FeedItem id will be changed by the selected action.
  • evt.data.analysisObjectID : String
    • Which measure item was clicked.
  • evt.data.actionID : String
    • Which action was selected in the action list.
Example

Sample code:

var vizBuilder = new VizBuilder(...);
var actionChangedHandler = function(event) {
    console.log(event.data);
}
vizBuilder.addEventListener('actionChanged', actionChangedHandler);

vizTypeChanged

Dispatches "vizTypeChanged" event when visualization type changes by UI operation. 
type: "vizTypeChanged"
handler: function({
                   type: "vizTypeChanged",
                   data: {
                       oldType: "",
                       newType: ""
                   }
})
Properties
  • evt.type : String
    • "vizTypeChanged"
  • evt.data.oldType : String
    • vizType
  • evt.data.newType : String
    • vizType
Example

Sample Code:

var vizBuilder = new VizBuilder(...);
var vizTypeChangedHandler = function(event) {
    var log = "vizTypeChanged event from vizBuilder and viz type changed from " +
                 event.data.oldType  + "to " + event.data.newType + ".";
    console.log(log);
};
vizBuilder.addEventListener('vizTypeChanged', vizTypeChangedHandler);

addEventListener (type, listener, scope, priority)

Registers an event listener object with an EventDispatcher object so that the listener receives notification of an event. You can register event listeners on any EventDispatcher object for a specific type of event, scope, and priority. If you no longer need an event listener, remove it by calling removeEventListener(), or memory problems could result.
Parameters:
  • type : String
    • The type of event.
  • listener : function
    • The listener function that processes the event.
  • scope : Object
    • The scope.
  • priority : int
    • The priority level of the event listener.

feedAnalysisObjects (analysisObjects, append)

Update or add feed FeedItem.
Parameters:

    • actually applied feeds by BVR (Best Visualization Recommendation)

var currentAnalysisObjects = vizBuilder.feedAnalysisObjects();

feeds (value)

Get/Set all feeds data. To get feeds, please use feeds() or feeds({withMND: true/false}). The feeds order is always same with the feeding definition of chart. If the user shows MND in UI, with option {withMND : true}, return current feeds with MND; with option {withMND : false}, return current feeds without MND; with no option, return current feeds. If the user doesn't show MND in UI, get feeds always return current feeds, the option value will be ignored. To set feeds, please use feeds(feedItems).
Parameters:
  • value : Array.<sap.viz.controls.common.feeds.FeedItem> | JSON
Returns:
  • Array<sap.viz.controls.common.feeds.FeedItem>
    • value
Example:

Sample code:

var vizBuilder = new VizBuilder(...);
//Set feeds, please refer to FeedItem for details.
var newFeeds = [
    new FeedItem("primaryValues", "Measure", [
        new AnalysisObject("M1", "M1", "measure", "number")]),
    new FeedItem("axisLabels", "Dimension", [
        new AnalysisObject("D1", "D1", "dimension", "string")]),
    new FeedItem("regionColor", "Dimension", [
        new AnalysisObject("D2", "D2", "dimension", "string")])
];
vizBuilder.feeds(newFeeds);
//Get feeds
var currentFeeds = vizBuilder.feeds();
