Namespace: RuleDefintion

The rule defintion, it's for custom renderer to match the context in customizations.
rules
SupportedType: ObjectArray
Description: Rules may have one or more rule objects. If one data point matches many rules, the last rule will *win.
dataContext
SupportedType: Object or ObjectArray
Description:
1 - For dimension , we support three kinds of operations (equal, notEqual, in).
Judgement priority: equal > notEqual > in. If you set {equal: "123", in: ["China"]}, only *the equal keyword will work.
equal:
1) You can use it directly without keyword equal.
example: dataContext: {Country: "China"}
2) You can use it with keyword equal
example: dataContext: {Country: {equal: "China"}}
notEqual:
example: dataContext: {Country: {notEqual: "China"}}
in:
example: dataContext: {Country: {in:["China","USA"]}}
2 - For measure or dimensions of numeric value type (such as time) , we support three kinds of operations (max&min, min, max).
Judgement priority: max && min > max > min.
max&min:
example: dataContext: {Age: {min: 10, max: 20}}
min:
example: dataContext: {Age: {min: 10}}
max:
example: dataContext: {Age: {max: 20}}
If you want to match the data that its measure is revenue, you can config it as following:
1) Use "measureNames" or "MeasureNamesDimension" keyword:
We support both keywords and "measureNames" takes higher priority if both are set.
(*)This approach is not recommended.
example: {dataContext: {MeasureNamesDimension: "Revenue"}}
2) Use *:
example: {dataContext: {Revenue: "*"}}
Specially, if you are using time dimension as a condition of dataContext, Timestamp value and *Date time string are
both valid and equivalent.
example: dataContext:[{Date: {min:1357051870000, max:"2014/05/23"}}]
Attention:
  • If you set many data context with Array, the result will be the union of all data context.
    example: dataContext: [{Country: {in: ["China","USA"]}}, {Revenue: "*"}]
    (i.e. The data context will match the data which country contains China and USA, or the *data which measure is revenue)
  • If you set many dimension and measure condition in one data context, the result will be the *intersection of all condition.
    example: dataContext: {Country: {in: ["China","USA"]}, Revenue: "*"}}
    (i.e. The data context will match the data which country contains [China, USA] and measure *is revenue)
