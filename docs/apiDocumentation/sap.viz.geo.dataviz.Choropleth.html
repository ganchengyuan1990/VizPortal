Class: Choropleth

Class: Choropleth

Extends: sap.viz.geo.dataviz.AbstractViz

Constructor

Choropleth (options)

Create a new choropleth visualization object. It will show polygons of geo locations specified by data and fill them with different colors by values in data.

Feeding Information: 

{
    location : FieldId1,
    color : FieldId2
}

Parameters:
  • options : Object [Optional]
    • the configuration object.
    Properties
    • startColor : Color [Optional]
      • Choropleth start color, default value:"#fac99e"
    • endColor : Color [Optional]
      • Choropleth end color, default value:"#703C10"
    • colorPalette : Array [Optional]
      • Set colorPalette for choropleth. If this array length greater than numOfSegments, we will ignore startColor and endColor. Default is ["#fAC99E","#f7A660", "#f48323", "#b7621a","#703c10" ]. Set to [] if you want to use startColor and endColor .
    • numOfSegments : Number [Optional]
      • Set Interval number for choropleth. value range[1~9], default value is 5
    • legendValues : Number [Optional]
      • Set Interval values for choropleth. number array in ascending order
    • mbcLegend : Object [Optional]
      • Set the MBC(MeasureValue-Based-Color) legend configuration.
      Properties
      • formatString : String [Optional]
        • Set the format pattern to be applied for numbers in the MBC legend
      • unitFormatType : String [Optional]
        • Set the type of format pattern to be applied for numbers in the MBC legend
    • dataLabel : Object [Optional]
      • the configuration object for data label.
      Properties
      • visible : Boolean [Optional]
        • whether enable data labels, false by default.
      • showColorValue : Boolean [Optional]
        • whether color value show in data label, true by default. It take effect only when visible is true.
      • showLocationName : Boolean [Optional]
        • whether geo location name show in data label, true by default. It take effect only when visible is true.
      • autoHide : Boolean [Optional]
        • when data label exceed polygon bound box, weather hide or truncate labels, false by default.
      • format : String | Object [Optional]
        • a string to indicate the format pattern of numbers in data label. It can be also an object keys of which are ids of measures to specify the patterns separately.
      • unitFormatType : String | Object [Optional]
        • a string to indicate the type of format pattern of numbers in data label. It can be also an object keys of which are ids of measures to specify the patterns separately.
    • hover : Object [Optional]
      • Properties
      • color : string [Optional] [Default: null]
        • Set hovered data point color effect. Four operation functions will be supported. - darken/lighten/desaturate/greyscale
    • selected : Object [Optional]
      • Properties
      • color : string [Optional] [Default: null]
        • Set selected data point color effect. Four operation functions will be supported. - darken/lighten/desaturate/greyscale
    • deselected : Object [Optional]
      • Properties
      • color : string [Optional] [Default: null]
        • Set deselected data point color effect. Four operation functions will be supported. - darken/lighten/desaturate/greyscale

Example

add a new choropleth layer

... //get map using viz api

var data = {
   "metadata" : {
     "fields" : [{
       "id" : "Location",
       "name" : "City",
       "semanticType" : "Dimension", 
       "dataType" : "String"
     }, {
       "id" : "Measure",
       "name" : "Sales",
       "semanticType" : "Measure"
     }]
   },
   "data" : [["Los Angeles", 30.12], ["Long Beach", 50.0],
       ["Washington", 40.32]]
 };
var feeding = {location: 'Location', color: 'Measure'};
var viz = new sap.viz.geo.dataviz.Choropleth({
 startColor : 'rgb(194,227,00)',
 endColor : 'rgb(115,192,60)',
  nullColor : 'rgb(204,204,204)',
  numOfSegments : 4,
  colorPalette:[],
  legendValues:[0,34,56,122,123,234]
 });

var ds = new sap.viz.api.data.FlatTableDataset(data);
var layer = new sap.viz.geo.dataviz.Layer({
  viz : viz,
  data : ds,
  feeding: feeding
});
map.addLayer(layer);
