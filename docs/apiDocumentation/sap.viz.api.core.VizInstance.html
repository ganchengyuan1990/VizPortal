[static] selection (selectionPoint, options)

Get/Set selected data points.
To get selection, please use selection() or selection(Options).
To set selection, please use selection(selectionPoint), or selection(selectionPoint, Options).
Parameters:
  • selectionPoint : Array [Optional]
    • Array of Objects with either data or ctx should be set to points. Each point should be the following: 
    {
    "data": {key: 'value', ...}, // (Optional) For the FlatTableDataset, the key is ID.
    "ctx":  [{mi: 0, ...}, {...}, ...], // (Optional) It takes higher priority when both parameters
                                        // of 'data' and 'ctx' are used.
}
  • options : Object [Optional]
    • Selection options 
    
Parameters used in getting data points from selection:
{
    "withDataCtx": Boolean, // (Optional) This parameter is only applicable to all viz charts, in which the ID starts with "viz/". The default value is false.
    "withInfo": Boolean, // (Optional) this parameter is deprecated.
}

Parameters used to set data points to be selected:
{
    "selectionMode": "inclusive" or "exclusive" // (Optional) The default value remains the same as the
                                                // selection mode set by chart property
                                                // of "interaction -> selectablity -> mode". It takes
                                                // higher priority when both parameters of
                                                // 'clearSelection' and 'selectionMode' are used.
    "clearSelection": Boolean // (Optional) This parameter is deprecated.
                              // parameter. Use 'clearSelection' for backward-compatibility only.
}
Returns:
  • Array | Boolean
    • Array of Points(for getting selection). Each point is 
    {
    "data": {key: 'value', ...},
    "ctx":  [{mi: 0, ...}, {...}, ...], // (Optional) Depends on "withDataCtx".
    "info":  [{type: 'additionalData', ...}, {...}, ...], // (Optional) Depends on "withInfo".
}
    or Boolean (for setting selection) 
        True means setting selection is successful.
    False means setting selection is unsuccessful.
Example:

Sample Code:

Get sample:
 VizInstance.selection(); this will get the selected data points.
 VizInstance.selection({withDataCtx: false}); this will get the selected data points without the data context.
 VizInstance.selection({withInfo: false}); this will get the selected data points without additional info.

[static] properties (props, options)

Get/Set chart properties
[NOTE: the options argument is only for info charts (chart type starts with "info/")]
When no arguments, return all properties.
When the first argument "props" is valid, merge it into existing user setting properties.
When options.level is valid (only "user" for now), return specific level properties.
When options.level is valid and options.replace is true, replace existing properties of this level with the input properties.
When there is only one argument or arguments are invalid, return this vizInstance; otherwise, return current properties (of specific level if options.level is valid) or empty object.
Parameters:
  • props : Object
    • Refers to Chart Property
  • options : Object [Optional]
    • extra options, only support "level" and "replace" for now. Only available for info charts
Returns:
  • Object
    • sap.viz.api.core.VizInstance
Example:

Sample Code:

// set properties (always user level)
VizInstance.properties({title: { visible : true, text :'Sample Bar Chart Title'}});
// get all properties
var props = VizInstance.properties();
// get user-set properties
var userProps = VizInstance.properties({}, { level: "user" });
// replace user properties
VizInstance.properties({ 'title.text': 'New Title' }, { level: "user", replace: true });
// reset user-set properties (replace with empty object)
VizInstance.properties({}, { level: "user", replace: true });

[static] feeding (feeding)

Get/Set data feeding information. To get feeding, use feeding()
To set feeding, use feeding(feeding)
Parameters:
  • feeding : Array | Object | null [Optional]
    • If passing in CrossTable dataset sap.viz.api.data.CrosstableDataset, the parameter should be an array of feeding items. 
    [{
  "feedId":feedId  // Feed ID for example: "regionColor"; refer to chartproperty
  "binding": [{
    "type" : "analysisAxis" | "measureValuesGroup" | "measureNamesDimension",
    "index":  Number
  }]
}]

For example:
[{
  "feedId" : "primaryValues",
    "binding" : [{
       "type" : "measureValuesGroup",
       "index" : 1
    }]
 }, {
  "feedId" : "regionColor",
     "binding" : [{
       "type" : "analysisAxis",
       "index" : 2
     }, {
       "type" : "measureNamesDimension"
     }]
 }, {
  "feedId" : "axisLabels",
    "binding" : [{
       "type" : "analysisAxis",
       "index" : 1
    }]
}]
    If passing in the FlatTable dataset sap.viz.api.data.FlatTableDataset, the parameter should be an object. 
    {
  "feedId": [metadata|"measureNamesDimension"]
}

For example:
{
  axisLabels: ["measureNamesDimension", "Year"],
  regionColor: ["Country"],
  primaryValues: ["Profit", "Revenue"]
}
    By passing a null feeding value in the CrossTable dataset, the chart will clean the feeding and use autofeed to create the chart.
Returns:
  • Array | Object
    • When it receives feeding, it returns the array of the CrossTable dataset sap.viz.api.data.CrosstableDataset or object of the FlatTable dataset sap.viz.api.data.FlatTableDataset feed items. When setting feeding, it returns the VizInstance sap.viz.api.core.VizInstance

[static] customizations (Customization)

Get/Set chart customizations
Parameters:
  • Customization : Object
    • settings with id, customOverlayProperties , customRendererProperties, customInteractionProperties.
    customRendererProperties structure: {customRendererID:{'rules':[{"dataContext":ruleCondition, "Properties":{}}...]}.
    RuleCondition of customRendererProperties can be an array or object, the definition can be referred to RuleDefintion.
    customRendererID: the id of BaseCustomRenderer.
    ruleCondition: the data context matched rule.
    properties: the properties for the matched context rule.
Returns:
  • Objet
    • Current customizations id , customOverlayProperties, customRenderProperties and customInteractionProperties.
Example:

Sample Code:

VizInstance.customizations({
      id: id,
      customOverlayProperties: {
          indicatorColor: "#FF0000"
      },
      customRendererProperties: {
         'custom.markerRenderer': {
            "rules": [{
               "dataContext": [{
                  'Year': {
                     'in': ['2001', '2002']
                  }
               }],
               "properties": {
                  fill: "blue",
                  stroke: "blue",
                  borderColor: "blue",
               }
            }, {
               "dataContext": {
                  Country: "USA",
                  Product: "Car",
                  Year: "2002"
               },
               "properties": {
                  fill: "red",
                  stroke: "red",
                 background: "red"
               }
            }]
         }
      },
      customInteractionProperties: {
          'custom.infobar.interaction': 4 {
              highlighter: "#0000FF"
          }
      );

[static] scales (scales)

[NOTE: This API is only applicable to all info charts, in which the ID starts with "info/"]
Setting/getting scales
When no arguments, return all scales.
When argument "scales" is valid, merge it into existing user setting scales.
When options.level is valid (only "user" for now), return specific level scales.
When options.level is valid and options.replace is true, replace existing sacels of this level with the input scales.
When there is only one argument or invalid arguments, return this vizInstance; otherwise, return current scales (of specific level if options.level is valid) or empty array
Parameters:
  • scales : Array [Optional]
    • Scale settings. 
    [
 {
   "feed": "valueAxis", // Feed named "valueAxis"
   "type": "linear", // Currently not working
   "min": 0, // Min value of the scale
   "max": 300 // Max value of the scale
 }, {
   "feed": "color", // Feed named "color"
   "type": "color", // Currently not working
   "palette": ["#00ffff", "#ff00ff", "#ff1122", "#ffff00", "#f99f99", "#99ff99"] // Color palette
 }
]
When setting the scales in multiple times, the last one will always override the former one if "feed" name is the same.
Returns:
  • Array
    • Scale settings of current vizInstance.
Example:

Sample Code:

// set scales (always user level)
VizInstance.scales([{ feed: "valueAxis", type: "linear", min: 0 }]);
// get all scales
var scales = VizInstance.scales();
// get user-set scales
var userScales = VizInstance.scales([], { level: "user" });
// replace user scales
VizInstance.scales([{ feed: "valueAxis", type: "linear", min: 0 }], { level: "user", replace: true });
// reset user-set scales (replace with empty array)
VizInstance.scales([], { level: "user", replace: true });
