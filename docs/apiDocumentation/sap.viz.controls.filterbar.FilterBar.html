Constructor

FilterBar (div, uiConfig)

FilterBar is a standard standalone UI control to display current FilterItems.
Parameters:
  • div : HTMLElement
    • The dom element to host the control.
  • uiConfig : JSON [Optional]
    • Configuration for the control. If without the uiConfig, use the default uiconfig to create the control.
         uiConfig.useBuildInFeedList:      whether using the built-in FeedList in our package.
     uiConfig.maxLines:                max lines of filters can be shown on the FilterBar, default value is 1.
     uiConfig.addButtonVisible:        show add button, default value is true.
     uiConfig.showInformationIcon:     specify whether show information icon for read-only filters, default value is false.
     uiConfig.informationIconTooltip:  tooltip for information icon.
     uiConfig.showClearIcon:           specify whether show clear icon, default value is false.
    Sample Code:
       // Get a default uiConfig of FilterBar from GlobalConfig. 
    var uiConfig = sap.viz.controls.common.config.GlobalConfig.defaultUIConfig(sap.viz.controls.common.config.GlobalConfig.DEFAULT_UICONFIG_TYPE_FILTERBAR);
    // Customize the default uiconfig.
    uiConfig.addButtonVisible = false;
    // Create a FilterBar control.
    var filterBar = new FilterBar($(div), uiConfig);

Events

requestFeed

Dispatches "requestFeed" event when click 'Filter' icon at top of the filter container and the flag useBuildInFeedList of uiConfig values false, which means application does not use Viz Container's built-in feedlist. Then application may need to pop up a customized feedlist, and dispatch a event with feed when a item selected.
Please ignore this event if using Viz Container's built-in feedlist.
type: "requestFeed"
handler: function({
                   type: "requestFeed",
                   data: {
                       fieldLocalPosition: {triggerWidth, x, y}
                   }
})
Properties
  • evt.type : String
    • "requestFeed"
  • evt.data.fieldLocalPosition : Object
    • "the data for calculating the location where the feed list will pop up at"
    Properties
    • triggerWidth : Object
      • "the width of the filter container"
    • x : Object
      • "the left position of the filter container"
    • y : Object
      • "the top position of the filter container"
Example

Sample code:

var filterBar = new FilterBar(...);
var feedRequestHandler = function(event) {
    //User needs to build up a feedlist
    var feedList = new FeedList(...);
    //User needs to add item selected event for feedlist and handle it
    feedList.addEventListener('itemSelected', function(event) {
        var selectedItem = event.data.selectedItem;
        // update filter base UI
        filterBar.filterItems(selectedItem);
    }, this);
    //User needs to calculate the position feedlist pop up at.
    var popupPosition = feedList.calculatePopupPosition(event.fieldLocalPosition);
    feedList.open(popupPosition);
}
feedList.addEventListener('requestFeed', feedRequestHandler);

requestAddFilter

Event: sap.viz.controls.common.constants.FilterEventConst.REQUEST_ADD_FILTER
When user clicks FilterButton and pick an AnalysisObject on the popup FeedList, FilterBar will notify host application to create a new filter item 
type: "requestAddFilter"
handler: function({
                   type: "requestAddFilter",
                   token: "",
                   data : {
                     analysisObject: AnalysisObject
                   }
})
Properties
  • evt.type : String
    • "requestAddFilter"
  • evt.token : Object
    • ""
  • evt.data.analysisObject : sap.viz.controls.common.feeds.AnalysisObject
    • AnalysisObject
Example

Sample Code:

var filterBar = new FilterBar(...);
var requestFilterHandler = function(event) {
     //create filter item base on event.data.analysisObject
     // update filter base UI
     filterBar.filterItems(newfilters);
};
filterBar.addEventListener('requestAddFilter', requestFilterHandler);

addEventListener (type, listener, scope, priority)

Registers an event listener object with an EventDispatcher object so that the listener receives notification of an event. You can register event listeners on any EventDispatcher object for a specific type of event, scope, and priority. If you no longer need an event listener, remove it by calling removeEventListener(), or memory problems could result.
Parameters:
  • type : String
    • The type of event.
  • listener : function
    • The listener function that processes the event.
  • scope : Object
    • The scope.
  • priority : int
    • The priority level of the event listener.
