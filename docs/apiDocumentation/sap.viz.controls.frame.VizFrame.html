Class: VizFrame

Extends: sap.viz.controls.common.VizControlHost

Constructor

VizFrame (div, uiConfig)

The VizFrame class is a composite control of following sub controls of Viz Container. VizFrame can be customized to show or hide each sub control.
    1. Chart: Host a CVOM visualization(i.e. Line Chart, Column Chart, Area Chart, Map) and visualization extension.
    2. ContextMenu: User can right click on the visualization or click the setting menu to display the context menu.
    3. PlayControl: User add animation feed and play the data frame-by-frame. Only Geo Bubble, Scatter and Bubble support this feature.
Parameters:
  • div : HTMLElement
    • The dom element to host the VizFrame control.
  • uiConfig : JSON [Optional]
    • Configuration for the control. If without the uiConfig, use the default uiconfig to create the control.
                  uiConfig.controls.contextMenu.enabled                           enable context menu or not, the default value is true.
                                                                              If enable context menu, when user right-click on the chart, user can do supported operation in the context menu dialog.
              uiConfig.controls.contextMenu.config.useCustomizedContextMenu   use customized context menu or not, the default value is false
                                                                              Only if enable context menu, this option works. User can follow the customized context menu specification to define their own context menu.
              uiConfig.controls.filterMenu.enabled                            enable filter menu or not, the default value is true
                                                                              If enable filter menu, when user select data points, user can filter/exclude the seelcted data points.
              uiConfig.controls.titleEditor.enabled                           enable title editor or not, the default value is true. 
                                                                              If enable title editor, when user mouse over to the title area, user can rename/reset the title text in the title editor.
              uiConfig.controls.playControl.enabled                           enable play control or not, the default value is true
                                                                              If enable play control, when user create a chart which support play control, user can set "dataFrame" feed and play the data frame-by-frame in the play control.
                                                                              If enable play control, VizFrame has UI5 dependencies: sap.ui.commons
              uiConfig.controls.dedicatedLegend.enabled                       enable responsive legend or not, the default value is false
    Sample code:
        // Get a default uiConfig of VizFrame from GlobalConfig.
    var uiConfig = sap.viz.controls.common.config.GlobalConfig.defaultUIConfig(sap.viz.controls.common.config.GlobalConfig.DEFAULT_UICONFIG_TYPE_FRAME);
    // Customize the default uiconfig.
    uiConfig.enableFilterMenu = false;
    // Create a VizFrame control.
    var vizFrame = new VizFrame($(div), uiConfig);

feedsChanged

Dispatches "feedsChanged" event when the feeding changes due to add/remove/change FeedItems on feeding panel
Application should listen to "feedsChanged" event to get the corresponding data and set it back to VizFrame to update the visualization with the new data. 
type: "feedsChanged"
handler: function({
                   type: "feedsChanged",
                   feeds: [{feedId, binding}]
})
Properties
  • evt.type : String
    • "feedsChanged".
  • evt.data : Object
Example

Sample Code:

var vizFrame = new VizFrame(...);
var feedsChangedHandler = function(event) {
    // an array of FeedItem objects
    var feedItems = event.data.feeds;

    // Application could restore to initial value, if there is no feeding
    if (feedItems || feedItems.length === 0) {
        var dataSet = new sap.viz.api.data.CrosstableDataset();
        dataSet.data(null);
        vizFrame.vizUpdate({
            data : null
        });
        return;
    }
    // Get filters if the application support it
    var filterItems = vizFrame.filterItems();

    // Application should handle some logic to get new data based on the feedItems and filterItems
    var newFeedData; // You can get data from database, data service, local data, etc.

    // update vizFrame
    var updateVizFrame = function(feedData) {
        var dataSet = new sap.viz.api.data.CrosstableDataset();
        dataSet.data(feedData);

        vizFrame.vizUpdate({
            'data' : dataSet
        });
    };
    upateVizFrame(newFeedData);
};
vizFrame.addEventListener('feedsChanged', feedsChangedHandler);

requestFilterValues

Dispatches the "requestFilterValues" event when double click the Filter object in FilterBar. The application may need to pop up a dialog to choose the filter values and call event.data.validate(true) after change. 
type: "requestFilterValues"
handler: function({
                   type: "requestFilterValues",
                   data: {
                        bounds, // an object contains left, top, width, height
                        filterName,
                        analysisObjectId,
                        filterItem, // an object of ValueFilter, DateRangeFilter or NumericRangeFilter
                        validate // function to validate filter item.
                    }
})
Properties
  • evt.type : String
    • "requestFilterValues".
  • evt.data : Object
    • It contains filterName, analysisObjectId and filterItem properties. User need to call validate(true) method if changed filterItem or call validate(false) if no change happens.
Example

Sample Code:

// listen to 'requestFilterValues' event
vizFrame.addEventListener('requestFilterValues', function(event) {
    var data = event.data;
    // Get current filter information
    var filterName = data.filterName;
    var analysisObjectId = data.analysisObjectId;
    var filterItem = data.filterItem;

    // 1. Application may pop up some filter value choose dialog.
    // 2. User input some new filter values.
    var values = 'USA,China';
    switch(filterItem.getType()) {
        case AbstractFilter.VALUE_TYPE:// display as 'USA,China'
            values = values.length>0 ? values.split(',') : [];
            var filterValues = [];
            for(var i = 0, length = values.length; i < length; i++) {
                filterValues.push({'value':values[i]}); // may contain more information. {'value': 'USA',
'key':'001'}
            }
            filterItem.updateValues(filterValues);
            break;
        case AbstractFilter.NUMERIC_RANGE_TYPE:// display as '10 - 20'
        case AbstractFilter.DATE_RANGE_TYPE:// display as '2000-12-1 - 2013-12-1'
            values = values.length>0 ? values.split(' - ') : [];
            if(values.length == 2) {
                filterItem.setMin(values[0]);
                filterItem.setMax(values[1]);
            }
            break;
    }

    // 3. Validate the filter item
    data.validate(true);
});

enableMorphing (value)

Enable or disable morphing. Note: The following viz types are not supporting morphing: - ChartConst.TYPE_GEO_BUBBLE - ChartConst.TYPE_GEO_CHOROPLETH - ChartConst.TYPE_GEO_PIE - ChartConst.TYPE_GEO_MAP - ChartConst.TYPE_CROSSTAB
Parameters:
  • value : Boolean
    • Default value is true. If value is true, enable morphing. If value is false, disable morphing.

exportToSVGString (option)

Export the current viz as SVG String. The viz is ready to be exported to svg ONLY after the initialization is finished. Any attempt to export to svg before that will result in an empty svg string.
Parameters:
  • option : Object [Optional]
    • { width: Number - the exported svg will be scaled to the specific width. height: Number - the exported svg will be scaled to the specific height. hideTitleLegend: Boolean - flag to indicate if the exported svg includes the original title and legend. hideAxis: Boolean - flag to indicate if the exported svg includes the original axis. }
Returns:
  • Object
    • the SVG Object of the current viz or empty svg if error occurs.

feeds (value)

Get/Set all feeds data. To get feeds, please use feeds() or feeds({withMND: true/false}). The feeds order is always same with the feeding definition of chart. If the user shows MND in UI, with option {withMND : true}, return current feeds with MND; with option {withMND : false}, return current feeds without MND; with no option, return current feeds. If the user doesn't show MND in UI, get feeds always return current feeds, the option value will be ignored. To set feeds, please use feeds(feedItems). Currently setting feeds for multilayer viz is not support.
Parameters:
  • value : Array.<sap.viz.controls.common.feeds.FeedItem> | JSON
Returns:
  • Array.<sap.viz.controls.common.feeds.FeedItem> | Array.<Array.<sap.viz.controls.common.feeds.FeedItem>>
    • Current feed items. An array of sap.viz.controls.common.feeds.FeedItem | An array of array of sap.viz.controls.common.feeds.FeedItem.
